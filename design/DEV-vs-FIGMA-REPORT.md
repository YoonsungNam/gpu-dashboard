# 개발 화면 vs Figma 시안 — 차이 리포트 (디자인팀 전달용)

작성일: 2026-06-11 · 기준 시안: `20260610-화면공유` v2 + 2026-06-11 디자인팀 수정분(v3)
대상 화면: Overview · GPU 활용 현황 · 토큰 활용 현황 (+ 공통 셸)

이 문서는 **의도적으로 시안과 다르게 구현한 항목**을 화면별로 정리한 것입니다.
각 항목에 변경 사유를 표기했습니다. 사유 분류는 아래 5가지입니다.

| 코드 | 사유 분류 |
|---|---|
| 운영피드백 | 운영(사용자) 측 피드백으로 확정된 변경 |
| 정합성 | 화면 간/화면 내 숫자 정합성 확보를 위한 데이터 구조 변경 |
| 운영정책 | 운영자가 수시로 바꾸는 정책값 → 설정 파일로 분리, UI는 파생 |
| 시안보완 | 시안에 정의되지 않은 동작(상태·인터랙션)을 보완 구현 |
| 시안모순 | 시안 내부 모순/오타로 판단해 한쪽 기준 적용 (확인 요청) |

미해소 질문 목록은 `design/QUESTIONS-for-design.md`를 참조해 주세요.

## 4. 운영 정책의 설정화 (정책 ↔ UI 파생 구조)

운영 정책은 **`src/lib/gradePolicy.ts`의 `GRADE_POLICY` 단 한 곳**입니다. '임계 기준'과
'과제 등급 기준'은 별개 체계가 아니라 같은 규칙의 두 가지 표현입니다:
등급 태그(우수/저활용)는 규칙 전체(and/or 조합)로, 지표 색상은 그 규칙을 지표 단위로
평가(policyLevel)한 것입니다. 정책값 하나를 바꾸면 행 태그·등급 카운트·셀/배지 색·
탭 하단 범례·Overview 점검 캡션과 배지·펼침 상세 색·지표 정의 패널의 임계 기준
그리드(policyBands)가 전부 함께 갱신됩니다.
(이전의 별도 셀 색 임계표 utilThresholds는 2026-06-11 피드백으로 폐지 — 이원화 해소)
