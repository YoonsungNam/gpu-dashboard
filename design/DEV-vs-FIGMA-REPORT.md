# 개발 화면 vs Figma 시안 — 차이 리포트 (디자인팀 전달용)

작성일: 2026-06-11 · 기준 시안: `20260610-화면공유` v2 + 2026-06-11 디자인팀 수정분(v3)
대상 화면: Overview · GPU 활용 현황 · 토큰 활용 현황 (+ 공통 셸)

이 문서는 **의도적으로 시안과 다르게 구현한 항목**을 화면별로 정리한 것입니다.
각 항목에 변경 사유를 표기했습니다. 사유 분류는 아래 5가지입니다.

| 코드 | 사유 분류 |
|---|---|
| 운영피드백 | 운영(사용자) 측 피드백으로 확정된 변경 |
| 정합성 | 화면 간/화면 내 숫자 정합성 확보를 위한 데이터 구조 변경 |
| 운영정책 | 운영자가 수시로 바꾸는 정책값 → 설정 파일로 분리, UI는 파생 |
| 시안보완 | 시안에 정의되지 않은 동작(상태·인터랙션)을 보완 구현 |
| 시안모순 | 시안 내부 모순/오타로 판단해 한쪽 기준 적용 (확인 요청) |

미해소 질문 목록은 `design/QUESTIONS-for-design.md`를 참조해 주세요.

## 4. 운영 정책의 설정화 (정책 ↔ UI 파생 구조)

운영 정책에 따라 자주 바뀌는 값은 코드 두 곳에 모았고, 나머지 UI는 전부 여기서 파생됩니다.
디자인 시안의 고정 문구/고정 색 대신 **정책값 변경 → 화면 전체 자동 갱신** 구조입니다.

- `src/lib/gradePolicy.ts` — 과제 등급(우수/저활용) 판정 규칙 (용도별 or/and 조합)
  → 행 태그, 등급 카운트, Overview 점검 캡션, 배지 색 임계, 지표 정의 패널의 '과제 등급 기준' 표
- `src/lib/util.ts`의 `utilThresholds` — 태스크별 셀 색 임계
  → 테이블 셀/배지 색, 탭 하단 범례, 지표 정의 패널의 '임계 기준' 표

## 5. 미해소 — 디자인팀 확인 요청 (요약)

상세는 `design/QUESTIONS-for-design.md` 참조.

- **A-1** Unit 구성 3번째 컬럼 헤더: 'GPU Util'(시안) vs 'GPU AH'(현재, 오타로 판단)
- **A-2** 펼침 상세 패널 스펙이 Overview/자원 화면에서 상이 — 의도된 분기인지
- **A-3** 저활용(빨강) 배지 배경 #FFE3E1 vs #FFD9D7 — 통일 색상 여부
- **A-4** 점검 카드 'N Projects' 카운터 색 — 액센트 vs 중립
- **B-2** 토큰 그룹 52개 표시 방식 — 전체 스크롤(현재) vs 페이지네이션
- **B-3(잔여)** 정렬은 구현 완료, **기본 정렬 기준**만 확정 필요 (현재 기본 미정렬)
- **B-6** 서비스 전체 모달 닫기 푸터 — 숨김(현재)이 맞는지
- **C-1** Overview 섹션 헤더 아이콘 실에셋 / **C-2** 워드마크 서체 에셋 / **C-3** 토큰 범례 말줄임 규칙 / **C-4** '42112' 콤마 표기 여부
